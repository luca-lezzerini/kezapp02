This is the updated, quick to seek, recording of the availability of a resource.<br/>It can have three states:<br/><ul>
<li>available: can be reserved</li><li>unavailable: cannot be reserved</li><li>reserved: was available and has been reserved from someone</li></ul>
This entity is managed with a cache strategy, i.e. it is a materialization of information derived from Calendar and more. If something changes, it is updated (if it will note require too much time) or it will be nightly, through a specific scheduled service, or it will be generated at request moment (i.e. if someone will ask for it, it will be calculated).<br/>The cacheKey field has a unique value which is, practically, an hash code with no conflicts probability.<br/>Its fields will be defined in successive UML updates<br/>
