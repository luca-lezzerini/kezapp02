The system (daily) or the user upon request, causes the regeneration of all due dates for any expiration period association. Only dates with solved to false must be regenerated.<br/>So, the algorithm is:<br/><ol>
<li>Programmatically remove all expiration dates with solved equal to false and changed equal to false</li><li>Programmatically select all expiration associations</li><li>For each of them, calculate the last expiration date selecting the one with the highest date. If it has changed true and solved false, skip this association expiration and go to next one</li><li>If it has changed = true and solved true or  changed field equal to false and solved = true create new Expiration date as step 5</li><li>If step 4 is ok, take the resolution date field as start date and sum the number of expiration days of the selected expiration period (other side of expiration association): this will be the new due date (i.e. expirationDate field). Create a new ExpirationDate and set its expirationDate to the new due date. Set changed and resolved to false, save it on DB associating to expiration period.</li><li>If no record with maximum date is present (i.e. no record at all of ExpirationDate) act as previous step creating a new Expiration record with start date equals to lastEventDate of expiration association. If this date is null, please log into ErrorMessages the information about no last event date available with information if it is asset or person, its code, the expiration period description.</li></ol><p/>
