Appears In: Expiration management
This UC allows user to manually change the selected due date. Once changed, the changed flag is set forever.<br/>The UC is used in those cases when the calculated expirationDate has to be changed for some reason and will not maintain the predefined rule (e.g. last event + predefined number of days). So the user changes the expiration date for a given asset or person.<br/>

Basic Path Basic Path
Details:
User selects the person or asset to be changed. The system shows its expiration dates. The user selects one of them and changes it. When changing it, a reason for change is manually written. This will cause a ForcedLog record to be added with the old expiration date (i.e. the one which has been changed), the new expiration date, the reason (i.e. a text description of why the date has been changed) and a timestamp of the instant when the change happened. Once changed the date, the changed field will remain always in true state. Records are added to ForcedLog at any change and cannot be removed.<br/>
View forced log Alternate
Details:
User selects a person or an asset and then select an expiration date. Clicking on the show history button, the system displays the list of records in forced log. <br/>
